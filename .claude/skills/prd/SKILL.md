# PRD

Write a product requirements document.

## Process
1. Gather context from the user about the feature/project
2. Research existing code and patterns that are relevant
3. Write the PRD with these sections:
   - Problem statement
   - Goals and non-goals
   - User stories / use cases
   - Technical requirements
   - Open questions
4. Save to `.claude/plans/prd-[name].md`

## Rules
- Focus on the "what" and "why", not the "how"
- Be explicit about non-goals to prevent scope creep
- Flag assumptions that need validation
